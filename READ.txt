Для запуска проекта необходимо открыть папку first-lar в phpStorm. После чего запустить сервер командой php artisan serve.
Функционал поставщика содержит регистрацию нового оборудования и получения отчетов.
Функционал администратора содержит получение отчетов по новому зарегистрированному оборудованию, перемещение его по складам
и получение отчетов по перемещенным оборудованиям.
Проверка функционала приложения будет описана для платформы Postman.

Для аутентификации поставщика необходимо использовать
логин: customer
пароль: 1234

Для администратора
логин: admin
пароль: 1111


Реализация функционала поставщика:

Для добавления товара необходимо выполнить пост запрос http://localhost/api/create-product.
В теле запроса указать параметры name, salary, serial_number, inventory_number.

Для получения отчета необходимо выполнить гет запрос http://localhost/api/report/customer-product.

Реализация функционала администратора:

Для добавления склада необходимо выполнить пост запрос http://localhost/api/add-storage/{product}.
В теле запроса указать параметр storage_id.

Для получения отчета о новом зарегистрированном оборудовании необходимо выполнить гет запрос http://localhost/api/report/new-product.
Для получения отчета о перемещенном оборудовании необходимо выполнить гет запрос http://localhost/api/report/moved-product.


Для фильтрации отчетов необходимо указать параметры при запросе отчета:

name - поиск по имени;
serial_number - поиск по серийному номеру;
inventory_number - поиск по инвентарному номеру;

start_salary, end_salary - нижний и верхний порог цены для фильтрации;
start_reg_date, end_reg_date - нижний и верхний порог даты регистрации для фильтрации;
start_move_date, end_move_date - нижний и верхний порог даты перемещения для фильтрации;
status - статус перемещения;
